# МОДУЛЬ 0. ПОДГОТОВИТЕЛЬНЫЙ (1 неделя)

## Цель: подготовить рабочее окружение и базовые понятия

### Темы:
1. **Что такое искусственный интеллект, ML и LLM**
   - Разница между ИИ, машинным обучением и большими языковыми моделями
   - Как работает LLM: токены, вероятности, предсказания
   - Основные модели: GPT, Claude, Gemini, Llama, Mistral

2. **Что такое AI-агент**
   - Определение: автономная система, воспринимающая окружение и действующая в нём
   - Разница между AI-ассистентом и AI-агентом
   - Архитектура агента: Восприятие → Мышление → Действие

3. **Установка окружения**
   - Python 3.10+
   - VS Code или PyCharm
   - Jupyter Notebook / Google Colab
   - Git и GitHub
   - OpenAI API key, HuggingFace token

### Практика:
- Установить Python и создать первый скрипт
- Получить API-ключ OpenAI (бесплатный тир)
- Запустить первый запрос к GPT через API

### Ресурсы:
- [Python за 1 час (YouTube)]
- [Документация OpenAI API]
- [Google Colab для новичков]

# МОДУЛЬ 1. БАЗОВЫЙ УРОВЕНЬ: ПРОМПТ-ИНЖИНИРИНГ (2 недели)

## Цель: научиться эффективно общаться с LLM

### Темы:
1. **Структура промпта**
   - Роль (Role)
   - Контекст (Context)
   - Задача (Task)
   - Формат вывода (Output Format)
   - Ограничения (Constraints)

2. **Техники промпт-инжиниринга**
   - Zero-shot, Few-shot, Chain-of-Thought
   - ReAct (Reasoning + Acting)
   - Self-consistency
   - Tree of Thoughts

3. **Работа с API**
   - Запросы через `openai` библиотеку
   - Параметры: temperature, max_tokens, top_p
   - Streaming ответов
   - Обработка ошибок

### Практика:
- Создать 10 различных промптов для разных задач
- Реализовать чат-бота на Python
- Сделать Chain-of-Thought решение математической задачи

### Проект модуля:
**Персональный ассистент-консультант** — бот, который отвечает на вопросы по выбранной теме (маркетинг, программирование, кулинария) с заданным стилем общения.

# МОДУЛЬ 2. РАБОТА С ИНСТРУМЕНТАМИ (2 недели)

## Цель: научить агента использовать внешние инструменты

### Темы:
1. **Function Calling (Вызов функций)**
   - Концепция: LLM выбирает, какую функцию вызвать
   - Описание функций (JSON Schema)
   - Парсинг аргументов
   - Выполнение и возврат результатов

2. **Базовые инструменты**
   - Поиск в интернете (DuckDuckGo, Google Search API)
   - Калькулятор и математические операции
   - Работа с датой и временем
   - Чтение файлов и документов

3. **API как инструменты**
   - Погода (OpenWeatherMap)
   - Переводы (DeepL, Google Translate)
   - Новости (NewsAPI)
   - Биржевые данные (Alpha Vantage)

### Практика:
- Создать агента с 3 инструментами
- Реализовать Function Calling с валидацией
- Обработать цепочку вызовов (поиск → анализ → ответ)

### Проект модуля:
**Умный туристический агент** — бот, который ищет погоду, достопримечательности, рекомендует рестораны и строит маршрут по заданному городу.

# МОДУЛЬ 3. ПАМЯТЬ И СОСТОЯНИЯ (2 недели)

## Цель: агенты должны помнить контекст и учиться

### Темы:
1. **Виды памяти в агентах**
   - Краткосрочная (контекстное окно)
   - Долгосрочная (векторные базы данных)
   - Процедурная (навыки и шаблоны поведения)

2. **Векторные базы данных**
   - Введение в embeddings (векторные представления)
   - ChromaDB, Pinecone, Weaviate, Qdrant
   - Семантический поиск
   - Chunking (разбиение документов)

3. **Управление контекстом**
   - Sliding window
   - Summarization (сжатие истории)
   - RAG (Retrieval-Augmented Generation)
   - Хранение диалогов в SQLite/PostgreSQL

### Практика:
- Создать векторное хранилище с ChromaDB
- Реализовать RAG для ответов по загруженным PDF
- Сделать агента с памятью о предыдущих диалогах

### Проект модуля:
**Агент-консультант по документации** — система, которая загружает техническую документацию (PDF, веб-страницы) и отвечает на вопросы пользователя, ссылаясь на источники.

# МОДУЛЬ 4. ФРЕЙМВОРКИ И БИБЛИОТЕКИ (2 недели)

## Цель: освоить готовые инструменты для создания агентов

### Темы:
1. **LangChain**
   - Архитектура: Chains, Agents, Memory, Tools
   - Создание цепочек (LLMChain, SequentialChain)
   - Встроенные агенты (zero-shot-react, conversational)
   - LCEL (LangChain Expression Language)

2. **LangGraph**
   - Графовые workflow для агентов
   - Узлы и рёбра
   - Условные переходы
   - Состояния и циклы

3. **AutoGen (Microsoft)**
   - Мультиагентные системы
   - ConversableAgent
   - GroupChat
   - UserProxyAgent

4. **CrewAI**
   - Роли (Roles) и задачи (Tasks)
   - Команды агентов (Crew)
   - Процессы: последовательный, иерархический

### Практика:
- Создать 3 разных агента на LangChain
- Построить workflow в LangGraph
- Запустить мультиагентную дискуссию в AutoGen

### Проект модуля:
**Команда аналитиков CrewAI** — система из 3 агентов (исследователь, аналитик, писатель), которая исследует тему, анализирует данные и пишет отчёт.

# МОДУЛЬ 5. АВТОНОМНЫЕ И МУЛЬТИАГЕНТНЫЕ СИСТЕМЫ (2 недели)

## Цель: создавать сложные системы взаимодействующих агентов

### Темы:
1. **Планирование и рассуждение**
   - ReAct цикл (Reason → Act → Observe)
   - Планирование задач (Task Decomposition)
   - Reflection (самооценка агента)
   - Коррекция ошибок

2. **Мультиагентная архитектура**
   - Конкуренция vs кооперация
   - Роли и специализация
   - Коммуникация между агентами
   - Посредник (mediator) и оркестратор

3. **Продвинутые паттерны**
   - Map-Reduce (разделение и агрегация)
   - Supervisor pattern
   - Parallel execution
   - Human-in-the-loop

### Практика:
- Реализовать ReAct с нуля (без фреймворков)
- Создать систему голосования агентов
- Добавить Human-in-the-loop для критических решений

### Проект модуля:
**Система разработки ПО** — архитектор, кодер, тестировщик и ревьюер работают вместе над созданием программы по ТЗ пользователя.

# МОДУЛЬ 7. БЕЗОПАСНОСТЬ И ЭТИКА (1 неделя)

## Цель: ответственное создание агентов

### Темы:
1. **Prompt Injection и защита**
   - Direct и Indirect injection
   - Jailbreaking
   - Санитизация входных данных
   - System prompt hardening

2. **Безопасность выполнения**
   - Sandboxing (изолированное выполнение)
   - Разрешённые и запрещённые операции
   - Rate limiting
   - Мониторинг действий агента

3. **Этика и ограничения**
   - Прозрачность (пользователь должен знать, что общается с ИИ)
   - Конфиденциальность данных
   - Биасы моделей и их компенсация
   - Соответствие GDPR и другим регуляциям

### Практика:
- Провести тесты на prompt injection
- Реализовать sandbox для выполнения кода
- Создать систему логирования действий

# МОДУЛЬ 8. PRODUCTION И РАЗВЁРТЫВАНИЕ (2 недели)

## Цель: превратить прототип в работающий продукт

### Темы:
1. **Архитектура production**
   - Микросервисы vs монолит
   - Очереди задач (Celery, Redis, RabbitMQ)
   - Кэширование (Redis)
   - Логирование и мониторинг (Loguru, Sentry)

2. **Модели и стоимость**
   - Оптимизация токенов
   - Кэширование ответов
   - Fallback-модели (отказоустойчивость)
   - Self-hosting (Ollama, vLLM, Llama.cpp)

3. **Интерфейсы**
   - REST API для агента (FastAPI)
   - Веб-интерфейс (Streamlit, Gradio)
   - Docker-контейнеризация
   - Deploy (Railway, Render, AWS)

4. **Тестирование**
   - Unit-тесты для инструментов
   - Evals для LLM (.evals фреймворк)
   - A/B тестирование промптов
   - Нагрузочное тестирование

### Практика:
- Обёрнуть агента в FastAPI
- Создать Streamlit-интерфейс
- Задеплоить через Docker
- Написать evals для качества ответов

### Финальный проект:
**Production-ready агент** — полноценное приложение с веб-интерфейсом, базой данных, мониторингом и деплоем, решающее реальную задачу (например, автоматическая поддержка клиентов или анализ рынка).
