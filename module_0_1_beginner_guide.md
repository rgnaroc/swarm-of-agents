## ЧАСТЬ 1: КАК РАБОТАЕТ ИСКУССТВЕННЫЙ ИНТЕЛЛЕКТ

### 1.1 Что такое LLM (большая языковая модель)?

Представь, что LLM — это гигантская предсказательная машина. Она не "думает" как человек, а предсказывает следующее слово на основе контекста.

**Аналогия**: представь, что ты читаешь предложение "Кошка сидит на...". Твой мозг мгновенно предлагает варианты: "окне", "крыше", "диване". LLM делает то же самое, но использует огромное количество текста, на котором она обучена.

**Ключевые концепции:**

| Термин | Объяснение | Аналогия |
|--------|------------|----------|
| **Токен** | Часть слова (1 слово ≈ 1-2 токена) | Слоги в слове |
| **Контекстное окно** | Сколько текста модель видит одновременно | Поле зрения |
| **Temperature** | Креативность ответов (0-2) | Творческий риск |
| **Parameters** | "Знания" модели (миллиарды чисел) | Опыт человека |

### 1.2 Разница: AI-ассистент vs AI-агент

**AI-ассистент (ChatGPT):**
- Отвечает на вопросы
- Не может действовать во внешнем мире
- Ждёт запроса пользователя
- Пример: "Расскажи про погоду" → текстовый ответ

**AI-агент:**
- Воспринимает окружение
- Принимает решения самостоятельно
- Использует инструменты (поиск, API, файлы)
- Работает в цикле до достижения цели
- Пример: "Забронируй мне отель в Париже" → поиск → сравнение → бронирование

**Архитектура агента:**
```
┌─────────────────────────────────────────┐
│            ОКРУЖЕНИЕ                    │
│  (интернет, файлы, базы данных)         │
└──────────────┬──────────────────────────┘
               │
┌──────────────▼──────────────────────────┐
│         ВОСПРИЯТИЕ                      │
│  (получение данных из окружения)        │
└──────────────┬──────────────────────────┘
               │
┌──────────────▼──────────────────────────┐
│         МЫШЛЕНИЕ (LLM)                  │
│  - Анализ ситуации                      │
│  - Планирование                         │
│  - Выбор действия                       │
└──────────────┬──────────────────────────┘
               │
┌──────────────▼──────────────────────────┐
│         ДЕЙСТВИЕ                        │
│  (выполнение через инструменты)         │
└──────────────┬──────────────────────────┘
               │
         (повтор цикла)
```

```bash

```

### 2.3 Установка библиотек

```bash
pip install openai python-dotenv jupyter
```

### 2.4 Получение API-ключа OpenAI

1. Зайди на [platform.openai.com](https://platform.openai.com)
2. Зарегистрируйся (требуется телефон)
3. Перейди в API Keys → Create new secret key
4. Скопируй ключ (покажется только один раз!)

**Безопасность ключа:**
Создай файл `.env` в папке проекта:
```env
OPENAI_API_KEY=sk-твой-ключ-здесь
```

> ⚠️ **ВАЖНО**: Никогда не публикуй API-ключ! Это как пароль от банковской карты.

### 2.5 Первый запуск

Создай файл `first_request.py`:

```python
import os
from openai import OpenAI
from dotenv import load_dotenv

# Загружаем ключ из .env
load_dotenv()

# Создаём клиент
client = OpenAI(api_key=os.getenv("OPENAI_API_KEY"))

# Отправляем запрос
response = client.chat.completions.create(
    model="gpt-4o-mini",  # дешевая и быстрая модель
    messages=[
        {"role": "user", "content": "Привет! Расскажи, что умеешь?"}
    ]
)

# Выводим ответ
print(response.choices[0].message.content)
```

Запусти:
```bash
python first_request.py
```

**Поздравляю! Ты только что сделал первый запрос к ИИ через API.** 🎉

## ЧАСТЬ 3: ПРОМПТ-ИНЖИНИРИНГ

### 3.1 Структура идеального промпта

```
РОЛЬ: Ты — эксперт по [предмету]

КОНТЕКСТ: У нас есть [ситуация]. Мы работаем над [задача].

ЗАДАЧА: Создай [что нужно]

ФОРМАТ: Представь результат в виде [список/таблица/JSON/код]

ОГРАНИЧЕНИЯ:
- Используй простой язык
- Максимум 5 пунктов
- Без технических жаргонизмов
```

### 3.2 Техники промпт-инжиниринга

#### Zero-shot (без примеров)
Просто описываем задачу:
```python
prompt = """Переведи на английский: Привет, как дела?"""
```

#### Few-shot (с примерами)
Даём примеры перед задачей:
```python
prompt = """Переведи сленг на литературный язык:

Сленг: "Кринж"
Литературный: "Чувство неловкости, стыда"

Сленг: "Рофл"
Литературный: "Шутка, юмор"

Сленг: "Кайф"
Литературный:"""
```

#### Chain-of-Thought (цепочка рассуждений)
Заставляем модель "думать" вслух:
```python
prompt = """Реши задачу пошагово:

В магазине было 15 яблок. Привезли ещё 8, но продали 12. 
Сколько яблок осталось?

Решение:
Шаг 1: Было 15 яблок
Шаг 2: Привезли 8, стало 15 + 8 = 23
Шаг 3: Продали 12, осталось 23 - 12 = 11
Ответ: 11 яблок"""
```

**Почему это работает?** Модель лучше справляется с задачами, когда "рассуждает" последовательно, а не сразу даёт ответ.

### 3.3 Работа с параметрами API

```python
response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[{"role": "user", "content": "Расскажи историю"}],
    
    # temperature: 0 = строго, 2 = максимально креативно
    temperature=0.7,
    
    # max_tokens: максимальная длина ответа
    max_tokens=500,
    
    # top_p: альтернатива temperature (0.1 = консервативно, 1 = разнообразно)
    top_p=0.9
)
```

**Когда что использовать:**

| Задача | Temperature | Top_p |
|--------|-------------|-------|
| Факты, код, SQL | 0.0 - 0.3 | 0.1 - 0.3 |
| Переводы, резюме | 0.3 - 0.5 | 0.7 - 0.9 |
| Творческий текст | 0.7 - 1.0 | 0.9 - 1.0 |
| Мозговой штурм | 1.0 - 1.5 | 0.95 - 1.0 |

## ЧАСТЬ 4: СОЗДАНИЕ ПЕРВОГО ЧАТ-БОТА

### 4.1 Простой консольный бот

```python
import os
from openai import OpenAI
from dotenv import load_dotenv

load_dotenv()
client = OpenAI(api_key=os.getenv("OPENAI_API_KEY"))

# Системный промпт — задаёт "личность" бота
SYSTEM_PROMPT = """Ты — дружелюбный помощник по программированию. 
Объясняй простыми словами, используй примеры кода.
Если не знаешь ответ — честно признайся."""

print("🤖 Бот: Привет! Задай мне вопрос по программированию.")
print("(напиши 'выход' для завершения)")

# Храним историю сообщений
messages = [
    {"role": "system", "content": SYSTEM_PROMPT}
]

while True:
    # Получаем ввод пользователя
    user_input = input("\n👤 Ты: ")
    
    if user_input.lower() in ["выход", "exit", "quit"]:
        print("🤖 Бот: До встречи!")
        break
    
    # Добавляем сообщение пользователя
    messages.append({"role": "user", "content": user_input})
    
    # Отправляем запрос
    response = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=messages,
        temperature=0.7,
        max_tokens=1000
    )
    
    # Получаем ответ
    bot_reply = response.choices[0].message.content
    
    # Добавляем ответ в историю
    messages.append({"role": "assistant", "content": bot_reply})
    
    # Выводим
    print(f"\n🤖 Бот: {bot_reply}")
```

### 4.2 Бот с контекстом (помнит тему)

```python
class ContextBot:
    def __init__(self):
        self.client = OpenAI(api_key=os.getenv("OPENAI_API_KEY"))
        self.messages = [
            {"role": "system", "content": "Ты — эксперт по Python. Объясняй с примерами."}
        ]
        self.max_history = 10  # храним последние 10 сообщений
    
    def ask(self, question: str) -> str:
        self.messages.append({"role": "user", "content": question})
        
        # Удаляем старые сообщения, кроме system
        while len(self.messages) > self.max_history + 1:
            self.messages.pop(1)  # удаляем самое старое (кроме system)
        
        response = self.client.chat.completions.create(
            model="gpt-4o-mini",
            messages=self.messages,
            temperature=0.5
        )
        
        answer = response.choices[0].message.content
        self.messages.append({"role": "assistant", "content": answer})
        return answer

# Использование
bot = ContextBot()
print(bot.ask("Что такое list comprehension?"))
print(bot.ask("А как сделать то же самое с условием?"))  # помнит контекст!
```

## ЧАСТЬ 5: ФОРМАТИРОВАНИЕ ВЫВОДА

### 5.1 Получение структурированных данных (JSON)

```python
import json

prompt = """Проанализируй текст и верни результат в формате JSON.

Текст: "Иван купил 3 яблока за 50 рублей в магазине 'Фрукты' 15 января"

Формат ответа (ТОЛЬКО JSON, без пояснений):
{
    "subject": "кто совершил действие",
    "action": "что сделал",
    "object": "что",
    "quantity": число,
    "price": число,
    "currency": "валюта",
    "location": "где",
    "date": "когда"
}"""

response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[{"role": "user", "content": prompt}],
    temperature=0.1  # низкая температура для стабильного JSON
)

result = response.choices[0].message.content
# Парсим JSON
data = json.loads(result)
print(f"Покупатель: {data['subject']}")
print(f"Потратил: {data['price']} {data['currency']}")
```

### 5.2 Markdown форматирование

```python
prompt = """Создай README.md для Python-проекта 'todo-app'.
Используй markdown-разметку: заголовки, списки, блоки кода."""

response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[{"role": "user", "content": prompt}]
)

# Сохраняем в файл
with open("README.md", "w", encoding="utf-8") as f:
    f.write(response.choices[0].message.content)
```
